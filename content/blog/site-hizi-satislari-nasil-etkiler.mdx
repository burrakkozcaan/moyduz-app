## Site Hızı ve Satış İlişkisi: Rakamlar

Yavaş site sadece kötü kullanıcı deneyimi değil, direkt gelir kaybıdır.

**Araştırmalar ne diyor:**
- Amazon: Her 100ms gecikme → %1 satış kaybı
- Google: 1→3 saniye yükleme → %32 daha fazla hemen çıkma
- Deloitte: %0.1 saniye hız iyileştirmesi → %8.4 dönüşüm artışı
- Walmart: 1 saniye iyileştirme → %2 dönüşüm artışı

**Türkiye ortalaması:** E-ticaret sitelerinin %60'ı mobilde 4 saniyenin üzerinde yükleniyor.

**Araçlar:**
1. **Google PageSpeed Insights** (ücretsiz, en önemli)
2. **GTmetrix** (detaylı analiz)
3. **WebPageTest** (gerçekçi test)
4. **Core Web Vitals** (Google Search Console)

- 50-89: Geliştirilmeli
- 0-49: Yavaş (acil aksiyon)

## Site Hızını Yavaşlatan Faktörler

### 1. Optimize Edilmemiş Görseller
E-ticaret sitelerinin en büyük hız sorunu. 5MB boyutunda ürün görseli kimseyi bekletemez.

**Çözüm:** WebP format + sıkıştırma + lazy loading

### 2. Ağır JavaScript

Eklentiler, canlı destek widget'ları, reklam scriptleri, chatbot'lar — her biri ek yükleme zamanı ekler.

**Çözüm:** Script optimizasyonu, defer/async yükleme

### 3. Yavaş Hosting

Ucuz paylaşımlı hosting ile hızlı site olmaz.

**Çözüm:** VPS, dedicated server veya cloud hosting (Cloudflare, Vercel, AWS)

### 4. Veritabanı Sorguları

Binlerce ürünlü katalogda veritabanı sorguları yavaşlatabilir.

**Çözüm:** Query optimizasyonu, caching, CDN

### 5. Hazır Platform Şişkinliği

Bazı platformların temaları gereksiz kodu beraberinde getirir.

**Çözüm:** Tema optimizasyonu veya [özel e-ticaret sistemine](/ozel-e-ticaret) geçiş

## Pratik Hız İyileştirme Adımları

**Acil (bu hafta):**
- Tüm görselleri TinyPNG ile sıkıştırın
- Google PageSpeed raporundaki "kritik" önerileri uygulayın

**Kısa vadeli (bu ay):**
- WebP formatına geçin
- CDN kullanmaya başlayın
- Gereksiz eklentileri kaldırın

**Uzun vadeli:**
- Hosting yükseltin
- [Platform migration](/ecommerce-migration) değerlendirin
